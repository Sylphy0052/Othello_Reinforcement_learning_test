# Documentation

ドキュメントを整備するコマンド。

## 🎯 Objective

<goal>
Ops & Writer の視点で、コードの変更をドキュメント（README, Docstring）に反映させること。
</goal>

## 📋 Instructions

1. **Sync:** コードの実装と `README.md` の記述に乖離がないか確認する。
2. **Docstring:** Google Style Guide に基づき、関数の説明・引数・戻り値を記述する。
3. **Changelog:** 最近の変更内容をリリースノート形式でまとめる。

## 🚫 Constraints

<rules>
- 専門用語を使わず、中学生でも読める平易な表現を心がけること。
- インストール手順や実行コマンドは、コピペで動くことを確認すること。
</rules>

## 📤 Output Format

- 修正対象ファイル (`README.md` 等) の差分。
